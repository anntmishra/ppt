Problem Statement

Given:

  • A permutation of numbers from 1 to N
  • We need to sort it in ascending order
  • Only allowed operation: swap elements
  • Cost of a swap = |i-j| (absolute difference of positions)

Goal: Find the minimum total cost to sort the permutation.

Example

Input: [3, 1, 4, 2]

3
1
4
2

Output: Minimum cost = 4

Key Observation: Cycle Decomposition

Any permutation can be decomposed into cycles:

  • A cycle follows where each element should go
  • Each element wants to be in its "correct" position (its value)
  • Following this path forms cycles
  • Each cycle can be sorted independently

For example, in [3, 1, 4, 2]:

  • Position 1 has value 3, which should be at position 3
  • Position 3 has value 4, which should be at position 4
  • Position 4 has value 2, which should be at position 2
  • Position 2 has value 1, which should be at position 1

This forms a cycle: 1 → 3 → 4 → 2 → 1

Initial Approach

For each cycle, we identified two methods to sort it:

Method 1: Internal Cycle Sorting

Use the minimum element in the cycle as a "base" and swap other elements with it:

  • Identify the minimum element in the cycle
  • Swap it with each other element in the cycle
  • Cost = sum of all distances between the min element and others
Method 2: Global Minimum Helper

Sometimes using the global minimum value as a helper is cheaper:

  • Use the global minimum as an intermediate element
  • Requires additional swaps but might have lower total cost
  • Especially useful when the cycle's minimum is far from other elements

Our solution: Calculate the cost of both methods for each cycle and pick the minimum.

Cycle: 1 → 3 → 4 → 2 → 1 (positions)

Values in cycle: 3, 4, 2, 1

Solution Idea

Our solution follows these key steps:

  1. Cycle Identification - Decompose the permutation into cycles by following elements to their correct positions
  2. Cost Calculation - For each cycle, calculate the cost of two methods:
    • Internal sorting with cycle's minimum element
    • Using global minimum as a helper
  3. Method Selection - Choose the method with minimum cost for each cycle
  4. Result - Sum up the costs of all cycles to get the final answer

Time & Space Complexity: O(N) - We process each element exactly once
